# Agent Guidance Protocol v2

## Purpose

OpenPanels exposes one Agent protocol entry point: `agent bootstrap`. The
installed CLI is the sole authority for the current Project, visible panel,
selection, capabilities, guides, skills, and persistent operations.

The separately installed `myopenpanels` entry Skill only detects or installs the
native CLI, starts or reuses Studio, opens `browserUrl`, and requests Bootstrap.
It must not duplicate panel workflows or CLI parameters.

## Communication Model

Communication remains local and pull-based:

```text
Agent -> current CLI -> SQLite/files
Studio -> local server -> SQLite/files
```

The CLI does not push prompts into the Agent. Before starting a new panel task,
the Agent reads Bootstrap and matches the user request against the returned
`applicableGuides.loadWhen` rules.

## Bootstrap Contract

`agent bootstrap --format json` returns:

- `protocolVersion` and `cliVersion`.
- `entrySkill` with the minimum MyOpenPanels Skill version, canonical source,
  and the startup-only conditional update instruction. A missing version is
  treated as outdated; an equal or newer version is left unchanged.
- `studioBinding` for the effective local Studio context.
- `focus` with `focusRevision`, Project, panel id, and panel kind.
- current Wiki, Canvas, task, and explicit-selection summaries.
- the complete current `capabilities` manifest.
- `applicableGuides`, `availableGuides`, and `availableSkills`.
- persistent `activeOperations` owned by the current Agent context.
- `suggestedCommands` and `nextRequiredAction`.

`focusRevision` identifies the state used when a new operation begins. It does
not prevent an existing operation from completing after the user switches
panels or Projects.

## Capabilities And Guides

Every capability contains a stable intent, current command template, argument
schema, target, `relatedGuides`, `preconditions`, and optional
`workflowIntent`. The manifest is generated by the current CLI binary.

Long or conditional workflows live in `agent-resources/guides` and ship inside
the CLI. A generation begin command returns the rendered required Guide so the
Agent does not need to rely on remembered instructions.

## Persistent Operations

Canvas image generation and Wiki document generation use persistent Operations:

1. Begin against the current focus.
2. Capture the original Project, panel, selection or document version.
3. Create visible generating state.
4. Let the user switch panels or Projects freely.
5. Complete, fail, or cancel against the captured target by Operation id.

Operations survive CLI and Studio restarts. They use `active`, `completed`,
`failed`, and `cancelled` states and are inspectable through the Agent operation
capabilities.

Canvas reference generation requires an explicit selection and never silently
uses fallback imagery. Completion requires generation metadata and replaces the
captured placeholder, or inserts into safe space if the placeholder was removed.

Wiki updates capture `contentVersion`; a concurrent edit returns
`content_conflict` rather than overwriting user content.

## Storage And Live Sync

Operations are stored in the `agent_operations` SQLite table. Target-bound
writes address explicit `sessionId + panelId` and never change the user's active
Project, active panel, or selection.

Only real panel-state revisions may reload the Canvas. Operation records and
other storage events must not rebuild the editor, clear selection, or increment
Canvas snapshot version.

## Compatibility Policy

The project has no public Protocol v1 compatibility surface. Removed Agent
entry points and aliases are not advertised or maintained. `wiki context`
remains a domain data-read command and is separate from the Agent protocol.

The stable product contracts are CLI installation, Studio start/reuse, and
`agent bootstrap`. All other commands are discovered from the current Bootstrap
response.

Skill freshness is checked only while processing the Bootstrap immediately
after Studio start or reuse. It is not a precondition for later panel commands
or an interruption mechanism for an active Operation. The Agent performs the
comparison because the CLI cannot reliably inspect every Agent host's Skill
installation directory.
