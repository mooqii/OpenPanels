# 维护 Karpathy LLM Wiki 中文版

当 Wiki 导航、摘要、索引或相关页面需要维护时，使用本参考流程。

工作流：

1. 读取 `wiki-conventions.md`，并查看可用的 Wiki 路径。
2. 读取 `SCHEMA.md`、`index.md`、`log.md`，以及影响导航或需要维护的页面。
3. 补齐缺失或不一致的基础页面。保留有用的现有结构，不要把它拍平成一层。
4. 按目录/类型更新 `index.md`，让每个仍有效的生成页面都有一个简短的相对 Markdown
   链接和摘要。
5. 必要时进行聚焦维护：修复失效或过时的链接，更新过时的溯源，标出未解决的矛盾，
   并把过长页面拆进既定目录层级。
6. 向 `log.md` 追加简短的 `maintenance` 记录，列出所有变更路径。

规则：

- 新生成或更新的索引摘要、分类说明和日志记录使用中文。
- 让 `index.md` 适合作为深入 Wiki 前首先读取的入口页。
- 摘要保持简洁、易扫读。
- 保留仍然有用的用户已有结构。
- 使用 `wiki-conventions.md` 中的目录层级、前置元数据、标签、溯源和链接约定。
- 不要在生成式 Wiki 里添加来源清单；提供的文档仍保留在生成式 Wiki 之外。
- 不要为了统一风格而重写所有页面。
- 不要因为切换到这个技能就翻译或重新生成整个 Wiki。

完成标准：

- 基础页面和索引/导航页反映当前生成式 Wiki 内容。
