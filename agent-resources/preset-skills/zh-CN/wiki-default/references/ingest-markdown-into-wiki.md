# 将 Markdown 来源整合进默认 Wiki

整合一份 Markdown 来源文档时，使用本参考。

执行步骤：

1. 阅读来源 Markdown。
2. 阅读 `wiki-conventions.md`，查看可用的 Wiki 路径，并检查 `SCHEMA.md`、`index.md`、`log.md` 以及可能需要变更的已有页面。
3. 第一次整合来源时，先建立 `SCHEMA.md`、`index.md` 和 `log.md`，再添加知识页面。如果尚未明确领域，就从来源推断一个范围较窄、便于日后调整的初始领域。
4. 对有价值的证据分类，在 `entities/`、`concepts/`、`comparisons/` 或 `summaries/` 下创建或更新聚焦的页面。
5. 在受影响页面中更新综合结论、出处、矛盾、限制条件、标签和有意义的相对 Markdown 链接。
6. 更新 `index.md`，确保每个新页面都能在正确分区被发现。
7. 在 `log.md` 追加一条简洁的整合记录，列出所有变更路径。

写作规则：

- 进行整合而不是倾倒内容，让每个来源都增强现有 Wiki 的知识网络。
- 优先创建能够跨来源持续积累的稳定页面。
- 只有当概念、实体、对比或总结具有持续价值时才创建新页面。某个主题即使只在一份来源中出现，只要它是该来源的核心，也可以独立成页。
- 不要创建来源页或文档镜像页；原始文档仍保留在生成 Wiki 之外。
- 新证据确认、细化、反驳或取代旧论断时，更新已有页面。
- 使用稳定的来源标识和必要的简短来源注释明确依据，不要把原始 Markdown 复制进页面。
- 遵循 `wiki-conventions.md` 中的目录与页面规范。
- 保持页面简洁、聚焦、易于导航，不要改写无关页面。
- 不要因为所选 Skill 变化而重新生成整个 Wiki。

完成标准：

- 来源中有价值的内容已被综合进 Wiki。
- 相关页面已经反映新证据。
- 当本次工作初始化或实质改变 Wiki 时，`SCHEMA.md`、`index.md` 和 `log.md` 均存在且保持一致。
