# 将源文档转换为 Markdown

当原始源文档需要先转换为 Markdown，才能整合进生成式 Wiki 时，使用本参考流程。

工作流：

1. 领取任务。
2. 检查源文档元数据和可用的原始文件。
3. 将源文档转换为干净的 Markdown。
4. 尽量保留标题、表格、列表、代码块、引用，以及有用的媒体或附件占位信息。
5. 用当前任务 ID 写入 Markdown 源文档。
6. 完成任务。

规则：

- 不要编造源文档中不存在的内容。
- 转换说明保持简短。
- 保留足够结构，方便后续从源内容中提取可长期复用的知识并整合进生成式 Wiki。
- 此任务只写入原始文档的 Markdown 表示，不要在这里创建生成式 Wiki 页面；后续的
  摄取任务负责该工作。
- 如果无法可靠完成转换，失败任务并给出清晰原因。

完成标准：

- 原始文档已有 Markdown 内容。
- 任务已完成，或已用可执行的错误信息标记失败。
